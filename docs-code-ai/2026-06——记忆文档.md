### 关键决策 / 坑点

- **祖先路径恢复的选择器代数**：CM6 行 DOM 扁平，"最近的深度 N 祖先" = `line-N:not(:has(~ line-N ~ .cm-active)):has(~ deeper.cm-active)`；同深度排除条件传递性地处理了浅层打断（A1 a2 B1 b2 c3 场景验证过）。各深度组的途径线选择器天然互斥，不依赖级联顺序。
- **不要直接 cap 任务行伪元素**：gist 原版 `.HyperMD-task-line::after { max-height }` 是全局污染（会影响其他功能画的伪元素）；改为在任务行上设 `--ouroboros-thread-*-max` 自定义属性、绘制规则里 `min()` 消费——自定义属性无副作用。
- **负 z-index 伪元素会被行背景吞掉**（BUG-007 教训）：`.cm-line` 非层叠上下文，负 z 伪元素沉到最底层；任何行背景（原生当前行高亮、引用底色）都会盖住它。装饰线正确层级是 `z-index: 0` + 标记元素提升到 1，而不是负 z。
- **`::before` 槽位冲突**：`active-line` 与 threading 在活动列表行共用 `::before`，threading 特异性更高胜出——登记 KNOWN-LIMITATIONS，不强行规避（两个伪元素槽都已占用）。
- **`border: <颜色token>` 是静默失效**（BUG-003 教训）：border 简写缺 style 时整条无效且无任何报错，必须配文本门禁。
- **master 与本分支状态分叉**：master 上存在 manifest 1.0.3 / package 1.0.2 / versions.json 缺条目的半完成版本升级（check 失败）；本分支版本一致。若日后合并需注意。

### 关键决策 / 坑点

- **`--line-width` 是死变量**：这是本轮最重要的发现。任何未来调阅读宽度的需求都必须经过 `--file-line-width` 绑定，否则不生效。已在 `check-theme.mjs` 加 `validateReadingWidth` 回归门禁，防止绑定被误删。
- **Live Preview 编号不能挂在 `#` marker 上**：Obsidian 在非编辑行隐藏 `.cm-formatting-header`，挂上去的内容会一起消失，必须挂 `.cm-line`。
- **JS-less 主题无法做运行时单选互斥**（B4）：只能靠确定性级联 + 文案，不要写「检测多选并警告」这种静态检查（主题无运行时状态可检测）。
- **minAppVersion 1.4.0 + 不降级**：< 1.4.0 的 app 不再被分发旧主题版本，而不是回退到旧版本。
