## 架构笔记

Ouroboros 是 Obsidian Theme，不是插件。模板中的 `PluginModule`、`ModuleManager`、`src/i18n` 等模式不能照搬；对应的可复用逻辑是：

- 用明确层级保持扩展性；
- 用文档体系保存任务、决策、bug、模式；
- 用检查命令作为完成门禁；
- 用 skeleton 文档帮助新会话快速恢复项目认知。

主题项目的核心架构应围绕：构建层、设计令牌层、核心 UI 层、内容/交互表面层、插件兼容与用户配置层。
